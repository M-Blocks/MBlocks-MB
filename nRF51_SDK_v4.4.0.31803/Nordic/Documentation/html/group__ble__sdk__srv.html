Detailed Description

All service modules have the same overall structure:

  • Init function: All services have an init function which will add the service and the corresponding characteristics to the stack data base. Configuration of optional characteristics, behaviour, data values etc. is passed to the init function using an init structure. For most services, a service structure is returned by the init function. To enable having several instances of the same service type in a single application, the service structure is allocated in the application, and a pointer to it is passed to the init function. To identify the service instance, a pointer to this service structure is passed to all other functions in the service API.
  • Stack event handler: Most services have a stack event handler (ble_xx_on_ble_evt()). It will react to events that are relevant to this service, and ignore all others. The handler must be called from the ble_stack_evt_handler callback function that is provided to the BLE Stack Event Handler by the application.
  • Service event handler: For most services, the application can/must supply a service event handler to the service during initialization. Using this handler, the service can notify the application about events in the service.
  • Service error handler: For some services, the application may supply a service error handler to the service during initialization. Using this handler, the service can notify the application about errors.
  • API functions: Most services have a set of API functions which the application can use to interact with the services.
